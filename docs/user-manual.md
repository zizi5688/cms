## 3. 页面/窗口级说明（逐页）

### 3.1 素材处理
页面目的:
将原始图片或视频做去印、超分、切宫格等处理，并把结果或视频任务上下文传给数据工坊。

控件清单（按钮/输入框/开关/菜单）:
- `选择文件`、`清空`、`打开源文件夹`
- 图片工具: `一键全流程`、`魔法去印`、`区域设置`、`画质重生`
- 视频工具: `截取当前帧为封面`、`上传封面`、`发送到数据工坊`
- 切分参数输入框: `行数`、`列数`
- 结果区按钮: `导出结果`、`发送到数据工坊`
- 弹窗: 预览弹窗、去印区域框选弹窗

操作步骤:
1. 点击 `选择文件` 导入图片或视频，或拖拽到投放区。
2. 图片模式可按需执行 `魔法去印`、`画质重生`、`开始切分`，或直接 `一键全流程`。
3. 视频模式可截帧或上传封面，然后点击 `发送到数据工坊`。
4. 切片结果可导出或一键发送到数据工坊目录。

关键功能使用示例（可直接照做）:
- 示例 A: `一键全流程`（去印 + 超分 + 切宫格）
1. 前置条件: 在“设置”页完成 `Python 路径`、`去印脚本路径`、`Real-ESRGAN 路径`配置。
2. 准备输入: 导入 6-20 张图片；建议先把 `行数=3`、`列数=3`（每张产出 9 张切片）。
3. 操作: 在素材处理页点击 `一键全流程`，等待日志从“步骤 1/3”到“步骤 3/3 完成”。
4. 预期结果: 原图目录下出现 `*_Clean`、`*_HD` 文件，且“切片结果”区域出现新宫格图。
5. 下一步: 点击 `发送到数据工坊`，在数据工坊直接读取该切片目录。
- 示例 B: `魔法去印`（仅去水印，不做超分和切片）
1. 前置条件: 已配置 `Python 路径` 与 `去印脚本路径`；建议先点 `区域设置` 框选水印区域并保存。
2. 准备输入: 导入一批有固定水印位置的图片（如同一机位截图）。
3. 操作: 点击 `魔法去印`，观察日志里每张图的处理开始/完成记录。
4. 预期结果: 同目录生成 `*_Clean` 图片；主预览区自动切换为处理后的文件集合。
5. 失败排查: 若提示“去印工具未配置”或“输出文件不存在”，回到设置页检查路径可执行性与脚本权限。
- 示例 C: `画质重生`（仅超分）
1. 前置条件: 已配置 `Real-ESRGAN` 可执行文件路径。
2. 准备输入: 导入低清图片（建议先做去印，再做超分，避免把水印也放大）。
3. 操作: 点击 `画质重生`，等待日志输出“全部完成”。
4. 预期结果: 同目录生成 `*_HD` 图片；点击缩略图可打开大图预览检查细节清晰度。
5. 失败排查: 若报 `exit != 0`，优先检查 Real-ESRGAN 路径、模型文件完整性、输入文件可读性。
- 示例 D: `开始切分`（仅切宫格）
1. 前置条件: 已导入图片素材（不要求必须先去印或超分）。
2. 准备输入: 设置 `行数` 和 `列数`（例如 `4 x 3`，表示每张输出 12 张）。
3. 操作: 点击 `开始切分`，等待日志提示“网格切片：生成切片 N 张”。
4. 预期结果: 原图同级目录生成 `SuperCMS_Batch_<时间戳>` 文件夹，内部为 `*_tile_<index>` 切片文件。
5. 下一步: 在“切片结果”区点击 `导出结果` 或 `发送到数据工坊` 继续编排任务。
6. 失败排查: 若报“切片尺寸无效”，通常是行列设置过大导致单格尺寸为 0，调小行列后重试。
- 示例 E: `视频封面`（截帧/上传）
1. 前置条件: 已导入一个视频文件，播放器能正常显示（或至少完成兼容处理）。
2. 操作路径 1（截帧）: 拖动视频到目标画面 -> 点击 `截取当前帧为封面`。
3. 操作路径 2（上传）: 点击 `上传封面` -> 选择本地图片文件。
4. 预期结果: 页面出现“已设置封面”预览，并在发送到数据工坊时随视频任务一起带入封面路径。
5. 下一步: 点击 `发送到数据工坊`，在数据工坊可看到“封面预览”状态。
6. 失败排查: 若提示“视频元信息尚未加载完成”，先播放 1-2 秒或稍等再截帧；若提示“请选择图片文件”，检查上传文件扩展名。

点击后系统反馈:
- 日志面板持续追加处理日志（`process-log` + 本地日志）。
- 成功会显示处理数量、输出路径或导出目录。
- 失败会在页面红字与日志中展示错误信息，并在部分场景弹出 `alert`。

异常与提示:
- 未配置 Python/脚本或 Real-ESRGAN 路径时阻止执行并提示。
- 输入路径无效、输出文件缺失、子进程非 0 退出时返回失败。
- 视频元信息未就绪时禁止截帧。

用例（正常流、异常流）:
- 正常流: 导入图片 -> 一键全流程 -> 生成切片 -> 发送到数据工坊。
- 异常流: 未配置本地工具 -> 点击处理按钮 -> 看到配置缺失提示并中断。

实现程度与证据:
- 判定: 已实现。
- 证据:
  - `src/renderer/src/components/modules/ImageLab.tsx` `handleAutoRun/startWatermarkRemoval/startHdUpscale/startSplit`
  - `src/main/index.ts` IPC `process-watermark`、`process-hd-upscale`、`process-grid-split`
  - `src/main/services/videoProcessor.ts` `prepareVideoPreview`
- 置信度: 高。

### 3.2 数据工坊
页面目的:
根据 CSV 与素材生成任务预览，支持批量选择并派发至账号队列。

控件清单:
- CSV 文本框
- 图片模式输入: 路径输入、`浏览`、`扫描`
- 分组参数输入: `分组数量`、`最大复用次数`、`每组最少/最多图片数`
- 操作按钮: `生成预览`、`重置`
- 任务区: 全选复选框、任务卡选择、底部派发浮层
- 派发浮层控件: 账号下拉、商品下拉、`派发至队列`

操作步骤:
1. 输入 CSV（标题/正文）。
2. 图片模式选择目录并扫描；视频模式由素材处理模块导入。
3. 点击 `生成预览` 检查任务卡。
4. 勾选任务，选择账号/商品，点击 `派发至队列`。

点击后系统反馈:
- 扫描完成后显示图片数量。
- 生成后刷新任务卡并写入状态管理。
- 派发过程中显示进度文案，完成后提示“已加入队列”。

异常与提示:
- CSV 为空或解析失败时终止生成。
- 账号未选时禁止派发。
- 派发失败时弹窗提示错误。

用例:
- 正常流: CSV + 图片目录 -> 生成预览 -> 选中派发 -> 媒体矩阵可见任务。
- 异常流: CSV 行数小于目标组数 -> 生成时报错。

实现程度与证据:
- 判定: 已实现。
- 证据:
  - `src/renderer/src/components/modules/DataBuilder.tsx` `handleScan/handleGenerate/dispatchSelected`
  - `src/renderer/src/lib/cms-engine.ts` `generateManifest/generateVideoManifest`
  - `src/main/index.ts` IPC `cms.task.createBatch` + 进度事件 `cms.task.createBatch.progress`
- 置信度: 高。

### 3.3 媒体矩阵（列表 + 日历）
页面目的:
统一管理账号、任务列表、日历排期、批量调度与删除。

控件清单:
- 账号区: 新增、刷新、登录、重命名、删除
- 视图切换: `日历` / `列表`
- 列表模式: 全选、阶段切换（待处理/已发布）、`批量排期`、`删除记录`
- 任务卡: 任务删除、失败重试、单任务排期修改/取消
- 弹窗: 批量排期弹窗、单任务排期弹窗
- 日历模式: 拖拽排期、任务详情、裂变任务（surprise remix）

操作步骤:
1. 选择账号或新增账号并登录。
2. 在列表视图进行批量勾选、排期、删除。
3. 在日历视图拖拽任务到目标日期时间。
4. 必要时同步商品。

点击后系统反馈:
- 账号和任务列表即时刷新。
- 排期成功后展示时间标签并写日志。
- 删除后从列表移除。

异常与提示:
- 无账号时阻断任务查看。
- 排期时间非法时提示并拒绝保存。
- 批量删除前有确认弹窗。

用例:
- 正常流: 选中 10 条待处理任务 -> 批量排期 -> 日历可见。
- 异常流: 无效 datetime 输入 -> 保存失败提示。

实现程度与证据:
- 判定: 已实现。
- 证据:
  - `src/renderer/src/components/AutoPublish/AutoPublishView.tsx` 账号/排期/删除/重试流程
  - `src/renderer/src/modules/MediaMatrix/CalendarView.tsx` 日历拖拽与批量排期
  - `src/main/taskManager.ts` `updateBatch/updateMany/listDueTasks`
  - `src/main/services/queueService.ts` `acquireNextTask/completeTask/failTask`
- 置信度: 高。

### 3.4 选品数据
页面目的:
导入采集 JSON，维护关键词并查看商品表格，支持导出 Excel。

控件清单:
- 顶部同步面板: `导入 JSON`
- 关键词栏: 关键词输入框、`+`、删除
- 商品表: 排序按钮组、`导出 Excel`

操作步骤:
1. 点击 `导入 JSON` 选择采集文件。
2. 在关键词栏新增或切换关键词。
3. 在商品表中排序并导出。

点击后系统反馈:
- 导入结果显示关键词/商品更新数。
- 历史同步记录展示最近 3 次。
- 导出成功后返回保存路径（控制台输出）。

异常与提示:
- 导入失败会显示错误消息。
- 无数据时显示空态提示。

用例:
- 正常流: 导入 JSON -> 选择关键词 -> 商品表排序 -> 导出 Excel。

实现程度与证据:

### 3.5 热度看板
页面目的:
对快照数据做趋势与潜力分析，支持加入待办、图搜 1688、候选供应商绑定。

控件清单:
- 左侧趋势雷达关键词列表
- 商品卡片网格 + Quick Look
- 底部操作: `搜同款`、`一键采购`、`复制链接`、`加入待办`
- 搜同款面板: 候选供应商列表、`绑定供应商`

操作步骤:
1. 选择关键词，浏览潜力商品卡。
2. 选中商品后可加入待办或发起搜同款。
3. 在候选供应商中选择并点击 `绑定供应商`。

点击后系统反馈:
- 状态文本区显示“加入成功/搜同款完成/需登录或验证码”。
- 缺失封面会触发异步抓图并回填。

异常与提示:
- 无快照、无关键词、无商品时均有空态提示。
- 1688 未登录或验证码时会触发专用事件提示。
- 调试模式返回 `DEBUG_MODE_ACTIVE` 时要求人工处理。

用例:
- 正常流: 选商品 -> 搜同款 -> 选择供应商 -> 绑定。
- 异常流: 无有效图片/目标售价 -> 搜同款失败并提示原因。

实现程度与证据:
- 判定: 部分实现。
- 证据:
  - `src/renderer/src/modules/HeatDashboard/index.tsx` `handleStartSourcing/handleBindSupplier`
  - `src/main/index.ts` IPC `IPC_SEARCH_1688_BY_IMAGE` + 1688 登录窗口
  - `src/main/services/scoutService.ts` `search1688ByImage/saveDashboardProductCover`
- 说明: `handleBindSupplier` 当前仅更新前端状态文案，未看到持久化提交接口调用。
- 置信度: 中。

### 3.6 设置
页面目的:
管理工作区、飞书配置、导入策略和本地工具路径。

控件清单:
- 工作区: 只读路径框、`切换工作区`
- 导入策略: `导入后删除源文件` 复选框
- 飞书配置: appId/appSecret/baseToken/tableId 与字段键输入、`测试连接`
- 工具配置: Real-ESRGAN、Python、脚本路径输入与 `浏览`

操作步骤:
1. 检查工作区并在必要时切换。
2. 填写飞书参数并测试连接。
3. 配置本地工具路径供素材处理模块调用。

点击后系统反馈:
- 成功切换工作区后提示并重启应用。
- 飞书测试成功会写入本地配置。
- 路径变更会自动触发 `save-config` 防抖保存。

异常与提示:
- 工作区不可写时显示 Debug 错误框。
- 飞书参数缺失时不允许测试。

用例:
- 正常流: 填写飞书参数 -> 测试连接成功 -> 素材模块可继续执行。
- 异常流: 工作区无写权限 -> 初始化失败提示。

实现程度与证据:
- 判定: 已实现。
- 证据:
  - `src/renderer/src/components/modules/Settings.tsx`
  - `src/main/services/workspaceService.ts`
  - `src/main/index.ts` IPC `workspace.*`、`feishu-*`、`get-config/save-config`
- 置信度: 高。

### 3.7 上传管理（隐藏入口）
页面目的:
把任务逐条同步到飞书多维表格，展示同步状态。

控件清单:
- `开始同步`、`重置`
- 进度条、统计卡片、任务状态列表

操作步骤:
1. 先在设置页完成飞书配置。
2. 在数据工坊生成任务。
3. 进入上传管理后点击 `开始同步`。

点击后系统反馈:
- 每条任务显示上传中/成功/失败。
- 成功返回 record_id。

异常与提示:
- 配置不完整时阻断上传。
- 图片上传失败时记录日志并继续下一张。

用例:
- 正常流: 批量任务上传 -> 状态更新为 success。
- 异常流: 飞书鉴权失败 -> 任务标记 error。

实现程度与证据:
- 判定: 部分实现（功能可用但默认入口关闭）。
- 证据:
  - `src/renderer/src/components/modules/UploadManager.tsx`
  - `src/renderer/src/components/layout/Sidebar.tsx`（`upload` 菜单项被注释）
- 置信度: 高。
