# Super CMS 功能使用手册（用户版）

## 简介

Super CMS 是一款面向飞书多维表格（Bitable/Base）的批量内容管理工具：你可以把一份 CSV 里的多条文案（标题/正文）与本地图片素材进行自动分组匹配，先生成“任务清单预览”，再一键顺序上传到飞书多维表格中，实时查看进度与结果。

适用场景：
- 批量生成图文内容并写入飞书 Base
- 将本地图片素材按规则分配到多条记录
- 用“预览 → 上传 → 复盘”的方式降低出错率

## 配置中心（Config）

Super CMS 需要调用飞书开放平台接口，因此你需要准备 AppID / App Secret，并填写你的 Base Token 与 Table ID。

### 1) 在飞书开放平台获取 AppID / App Secret

1. 打开“飞书开放平台控制台”，进入你要使用的应用。
2. 在应用的“凭证与基础信息”（或类似命名）页面，找到：
   - App ID（通常以 `cli_` 开头）
   - App Secret
3. 复制保存，稍后填写到 Super CMS 的“飞书配置”区域。

提示：
- App Secret 属于敏感信息，请不要发送给他人或截图公开。
- 需要确保应用具备访问多维表格/云文档相关能力（具体权限以你组织的开放平台配置为准）。

### 2) Base Token 与 Table ID 从哪里来

你需要知道目标多维表格（Base）的位置：
- Base Token（app_token）：多维表格应用的标识，通常形如 `bascn...`
- Table ID：Base 中某个数据表的标识，通常形如 `tbl...`

一般获取方式：
- 在多维表格页面中复制链接/分享信息，从 URL 中找到 `app_token` 与 `table_id` 字段（不同入口展示略有差异）。

### 3) “测试连接 (Test Connection)”按钮做什么

入口位置：
- 左侧导航进入「上传管理」页面
- 在“飞书配置”卡片底部点击“测试连接 (Test Connection)”

它会做两件事：
1. 校验凭证：用你填写的 App ID / App Secret 获取租户访问 token。
2. 校验表格：用 Base Token 与 Table ID 尝试读取该表（只读校验，确保配置可用）。

结果反馈：
- 成功：弹窗提示连接成功，同时控制台会输出 `[Feishu] 连接成功并已保存配置。`
- 失败：弹窗与控制台会提示失败原因（如凭证错误、无权限、Base/Table 填错等）。

## 数据工坊（Data Builder）

数据工坊用于两件事：
- 输入 CSV（标题/正文）
- 扫描本地图片文件夹并按规则分组，生成“任务清单预览”

### CSV 格式

CSV 要求：
- 标准 CSV（支持英文逗号分隔）
- 第一行是表头（Header）
- 至少包含两列：`Title` 与 `Body`（大小写不敏感；也支持 `标题` / `正文`）

推荐格式示例（含引号，适合正文里出现逗号、换行等复杂内容）：

```csv
Title,Body
"第一条标题","这里是正文第一段
这里是正文第二段（允许换行）"
"第二条标题","正文里可以有逗号，比如：A,B,C"
```

常见问题：
- 如果正文里包含换行或逗号，务必用英文双引号把整段正文包起来。
- 空行会被忽略；整行全空的记录不会生成任务。

### Folder Scan：扫描图片文件夹

操作步骤：
1. 点击“浏览 (Browse)”选择图片所在的文件夹。
2. 选择后会自动扫描；也可以点击“扫描 (Scan)”重新扫描。
3. 扫描完成后会显示“已载入图片：N 张”。

建议：
- 把要用的图片集中放在一个文件夹内，避免混入无关文件。
- 图片数量不足时仍可生成预览，但部分任务可能分配不到足够图片。

### Settings：分组与图片分配规则

1) 组数（0=按 CSV 行数）
- `0`：按 CSV 解析出来的有效行数生成任务（最常用）
- `N`：只取 CSV 前 N 行生成 N 组任务

2) 最大复用次数（Max Reuse）
- 同一张图片最多被分配到多少个任务里
- 值越大，图片越可能被重复使用；值越小，越“均匀但更容易不够用”

3) 最少/最多张数（Min Images / Max Images）
- 每个任务会在这个范围内随机分配图片数量
- 例如 Min=3、Max=5：每组会尝试分配 3～5 张图

当图片不足时：
- 系统会尽力分配，但如果某一组实际分配少于 Min，会在任务里出现提示（例如“图片不足：目标至少 3 张，实际分配 1 张”）。

### Preview：任务清单预览（Task List）

点击“生成预览 (Generate Preview)”后，你会在下方看到“任务清单预览”：
- 每一行代表一组任务（第 1 组、第 2 组…）
- 显示该组的标题摘要、已分配图片数量
- 下方缩略图用于确认图片分配结果

这份任务清单会被保存在应用的全局状态中，供后续“上传管理”使用。

## 上传管理（Upload Manager）

上传管理分为三部分：
- 同步控制区（Start/Reset + 进度条）
- 飞书配置（AppID/Secret/Base Token/Table ID + 字段映射 + 测试连接）
- 任务状态列表（每组任务的实时状态）

### Sync Process：开始同步

前置条件：
- 先在「数据工坊」生成任务清单预览
- 填好“飞书配置”与字段映射（Title/Body 必填，Image 可选）

同步步骤：
1. 点击“开始同步 (Start Sync)”。
2. 系统会按顺序处理每组任务：
   - 先上传该组图片（如果有分配图片）
   - 再创建多维表格记录（写入标题/正文/图片附件字段）
3. 你可以在进度条看到总体百分比，在底部控制台看到详细日志。

任务状态说明：
- Pending：尚未开始
- Spinner：正在上传/创建
- Check：成功（会显示 record_id）
- X：失败（会显示错误信息）

### Reset：重置

点击“重置 (Reset)”会清空本次会话相关内容，便于重新开始一轮任务流程：
- 清空当前任务清单与任务状态
- 清空 CSV 输入与已选择的图片文件夹
- 清空底部控制台日志

说明：
- 飞书配置输入通常会保留在界面中，方便你再次同步；如果你希望更换配置，直接修改输入框内容后重新“测试连接”即可。
