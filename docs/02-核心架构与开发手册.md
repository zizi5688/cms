## 一、快速上手与避坑

### 1.1 环境依赖

| 依赖 | 最低版本 | 说明 |
|------|---------|------|
| Node.js | 18+ | package.json 未声明 `engines`，但 Electron 39 要求 |
| Python 3 | 3.10+ | 仅开发环境水印功能需要（生产环境用 PyInstaller 打包） |
| ffmpeg / ffprobe | — | 通过 `ffmpeg-static` / `ffprobe-static` npm 包自动安装 |
| Real-ESRGAN | — | HD 超分辨率功能需要，macOS 版打包在 `AI_Tools/` 目录 |
| ADB | — | 代码中**未使用**，但如需调试安卓设备可能需要 |

### 1.2 首次启动

```bash
# 1. 安装依赖
npm install

# 2. 重编译 Native 模块 (关键！)
npm run rebuild

# 3. 启动开发服务器
npm run dev
```

### 1.3 编译避坑指南 (Critical)

#### better-sqlite3 重编译问题

**现象**: 启动后报错 `NODE_MODULE_VERSION mismatch` 或 `was compiled against a different Node.js version`

**原因**: `better-sqlite3` 和 `sharp` 是 C++ Native Addon。npm install 时会编译为系统 Node.js 版本的 ABI，但 Electron 用的是自己的 Node.js（版本可能不同）。

**解决方案**:

```bash
# 方案 1: 使用项目配置的 rebuild 脚本
npm run rebuild

# 方案 2: 手动 electron-rebuild
npx electron-rebuild -f -w better-sqlite3
npx electron-rebuild -f -w sharp
```

**原理**: `electron-rebuild` 会用 Electron 头文件重新编译 native 模块。

#### ASAR 解包配置

**位置**: `electron-builder.json`

```json
"asarUnpack": [
  "resources/**",
  "node_modules/sharp/**",
  "node_modules/@img/**",
  "node_modules/better-sqlite3/**",
  "node_modules/ffmpeg-static/**",
  "node_modules/ffprobe-static/**"
]
```

Native 模块必须从 ASAR 包中解压出来，否则 Node.js 的 `dlopen` 无法加载 `.node` 文件。如果新增 native 依赖，**必须**添加到此列表。

#### sharp 在 macOS ARM64 的问题

sharp 依赖 `@img/sharp-darwin-arm64` 等平台包。如果 `npm install` 在 x86 环境运行后拿到 ARM 机器上用，会报找不到 binding。解决：删除 `node_modules` 重新安装。

### 1.4 构建与打包

```bash
# 仅编译（不打包安装程序）
npm run build:app

# 完整构建（编译 + 生成安装包）
npm run build           # 当前平台
npm run build:mac       # macOS DMG
npm run build:win       # Windows NSIS
npm run build:linux     # Linux AppImage/deb/snap
```

**Python 引擎打包**（仅在需要水印功能时）:
```bash

```

#### 数据库文件位置

```

CREATE TABLE accounts (

```

⚠️ 当前主进程实际使用的是 JSON 版 AccountManager，此表**可能不是最新的**。

#### products 表

```sql
CREATE TABLE products (
  id        TEXT PRIMARY KEY,   -- 商品 ID (来自小红书)
  accountId TEXT NOT NULL,      -- 关联账号
  name      TEXT NOT NULL,      -- 商品名
  price     TEXT NOT NULL,      -- 价格 (字符串)
  cover     TEXT NOT NULL       -- 封面图 URL
);

CREATE INDEX idx_products_accountId ON products (accountId);
```

#### tasks 表 (核心)

```sql
CREATE TABLE tasks (
  id               TEXT PRIMARY KEY,              -- UUID
  accountId        TEXT NOT NULL,                  -- 关联账号
  status           TEXT NOT NULL,                  -- pending|processing|failed|publish_failed|draft_saved|scheduled|published
  mediaType        TEXT NOT NULL,                  -- image | video
  images           TEXT NOT NULL DEFAULT '[]',     -- JSON 数组: 图片路径列表
  videoPath        TEXT,                           -- 视频路径
  videoPreviewPath TEXT,                           -- 视频预览路径 (转码后)
  title            TEXT NOT NULL,                  -- 标题
  content          TEXT NOT NULL,                  -- 正文
  tags             TEXT,                           -- JSON 数组: 标签列表
  productId        TEXT,                           -- 关联商品 ID
  productName      TEXT,                           -- 关联商品名
  publishMode      TEXT NOT NULL,                  -- draft | immediate
  scheduledAt      INTEGER,                        -- 排期时间戳 (毫秒, NULL=未排期)
  publishedAt      TEXT,                           -- 发布时间 (ISO 字符串)
  createdAt        INTEGER NOT NULL,               -- 创建时间戳 (毫秒)
  errorMsg         TEXT NOT NULL DEFAULT '',        -- 错误信息
  errorMessage     TEXT,                           -- ⚠️ 遗留冗余字段，与 errorMsg 重复
  isRaw            INTEGER DEFAULT 0,              -- 是否为原始任务 (vs 裂变生成)
  -- 以下为队列扩展列 (ensureQueueColumns 动态添加):
  locked_at        INTEGER,                        -- 任务锁定时间戳
  retry_count      INTEGER DEFAULT 0               -- 重试次数
);

CREATE INDEX idx_tasks_accountId   ON tasks (accountId);
CREATE INDEX idx_tasks_status      ON tasks (status);
CREATE INDEX idx_tasks_scheduledAt ON tasks (scheduledAt);
CREATE INDEX idx_tasks_createdAt   ON tasks (createdAt);

-- 级联删除触发器
CREATE TRIGGER trg_accounts_delete_tasks
AFTER DELETE ON accounts
BEGIN
  DELETE FROM tasks WHERE accountId = OLD.id;
END;
```

**字段说明**:
- `images` 存储为 JSON 字符串 `["assets/images/abc.jpg", "assets/images/def.png"]`
- `tags` 存储为 JSON 字符串或 NULL
- `locked_at` 和 `retry_count` 通过 `ensureQueueColumns()` 动态 ALTER TABLE 添加（为兼容旧数据库）
- `errorMessage` 是遗留字段，新代码统一使用 `errorMsg`

### 2.3 SQLite 单例模式

**位置**: `services/sqliteService.ts:6-18`

```typescript
export class SqliteService {
  private static instance: SqliteService | null = null
  private db: any | null = null

  static getInstance(): SqliteService {
    if (!SqliteService.instance) {
      SqliteService.instance = new SqliteService()
    }
    return SqliteService.instance
  }

  private constructor() {}
}
```

- 经典单例，进程生命周期内只有一个连接
- `better-sqlite3` 是**同步 API**（不同于 node-sqlite3），所有 SQL 操作是同步的
- WAL 模式允许并发读，但写入仍然是串行的
- `better-sqlite3` 通过 **懒加载** (`import()`) 引入，在 `loadDatabaseCtor()` 中处理 ESM/CJS 兼容

### 2.4 配置存储

系统使用 `electron-store` 保存配置（非 SQLite），文件位于:
```
{userData}/config.json    # macOS: ~/Library/Application Support/super-cms/config.json
```

存储内容:
- `feishuConfig`: 飞书 API 凭证
- `realEsrganPath`: Real-ESRGAN 可执行文件路径
- `pythonPath`: Python 解释器路径
- `watermarkScriptPath`: 水印脚本路径
- `watermarkBox`: 水印区域坐标
- `defaultStartTime`: 默认排期开始时间 (如 "10:00")
- `defaultInterval`: 默认排期间隔 (分钟)
- `workspacePath`: 工作区路径
- `importStrategy`: 素材导入策略 (copy | move)

### 3.3 并发控制详解

```
层级 1: 心跳调度 (index.ts)
  runningScheduleByAccount: Map<string, Promise<void>>
  │  每 5 秒检查到期任务
  │  同一 accountId 不重复触发
  ▼

层级 2: IPC Handler (publisher.ts)
  runningByAccount: Map<string, Promise<...>>
  │  cms.queue.start 的去重
  │  同一 accountId 只允许一个 runner
  ▼

层级 3: PublisherService (publisher.ts)
  isPublishing: boolean
  │  实例级互斥锁
  │  如果已在发布，直接返回空结果
  ▼

层级 4: SQL 原子操作 (queueService.ts)
  UPDATE ... WHERE id = (SELECT ... LIMIT 1) RETURNING *
  │  SQLite 单写者保证原子性
  ▼

层级 5: GPU 限流 (index.ts)
  gpuLimit = pLimit(1)
  │  图片处理（超分/水印）串行执行
```

**注意**: 层级 1-3 都是内存锁，进程崩溃后全部丢失。只有层级 4 的 `locked_at` 字段持久化在数据库中，供下次启动时的僵尸任务恢复使用。

### 4.2 路径地狱 (Path Hell)

#### 问题: 跨平台迁移后图片全部丢失

**根因**: 旧版本在 tasks 表的 `images` 字段中存储了**绝对路径**，如 `/Users/alice/Desktop/photo.jpg`。换电脑后路径不存在。

**现有应对**:

1. **创建时相对化** (taskManager.ts): 新创建的任务会将图片复制到 `{workspace}/assets/images/` 并存储为 `assets/images/{hash}.jpg` 相对路径

2. **运行时回退** (index.ts safe-file 协议): 即使存的是绝对路径，safe-file 协议会通过文件名在 workspace 下搜索

3. **自动化脚本中的解析** (xhs-automation.ts): `resolveWorkspaceFilePath()` 会将 `assets/...` 格式的相对路径拼接上工作区路径

#### 问题: macOS 中文路径

**根因**: macOS 系统语言为中文时，`~/桌面/` 实际指向 `~/Desktop/`（Finder 显示本地化名称，但文件系统路径是英文）。

**现有应对**: `index.ts:405-422` 的 `tryResolveMacLocalizedPath()` 函数，维护了 6 对中英文映射。

#### 问题: Windows 路径分隔符

**现有应对**:
- `taskManager.ts:92` 的 `normalizeAssetRelativePath()` 将 `\` 替换为 `/`
- `xhs-automation.ts` 中的路径解析统一使用 `path.join()`
- safe-file 协议中的 Windows 路径解析 (`index.ts:369-383`) 处理了 `C:` 盘符

### 4.3 常见报错及处理

| 报错 | 原因 | 解决 |
|------|------|------|
| `NODE_MODULE_VERSION mismatch` | native 模块未对 Electron 重编译 | `npm run rebuild` |
| `[SqliteService] Failed to load better-sqlite3` | better-sqlite3 加载失败 | 检查 rebuild 是否成功；检查 ASAR 解包配置 |
| `[XHS] Not logged in: redirected to login page` | 小红书 Cookie 过期 | 手动通过登录窗口重新登录 |
| `[XHS] Automation timeout` | DOM 结构变化导致自动化失败 | 检查小红书后台是否改版，更新选择器 |
| `CRITICAL: Queue picked up an unscheduled task!` | 理论上不应出现 | 检查数据库中是否有 `scheduledAt=NULL` 但 `status='pending'` 的脏数据 |
| `[Asset] copy timeout` | 大文件复制超时 | 默认 20 秒超时 + 3 次重试，检查磁盘 IO |
| `[System] 异常中断：发布期间应用退出或崩溃` | 上次运行时崩溃 | 正常现象，崩溃恢复机制的标记信息 |

## 五、技术债务清单

以下是扫描代码后发现的技术债务，供后续开发参考：

### 高优先级

1. **双存储并存**: 账号和商品仍在 JSON (ElectronStore)，任务在 SQLite。`services/accountManager.ts` 已经写好了 SQLite 版本但未接入。应完成迁移。

2. **index.ts 过于庞大**: 1735 行的单文件包含了所有 IPC 注册、safe-file 协议、飞书 API、图片处理。需要拆分。

3. **遗留重复文件**:
   - `src/preload/xhs-automation.js` vs `src/main/preload/xhs-automation.ts`
   - `src/main/services/publisher.ts` vs `src/main/publisher.ts`
   - `src/main/services/accountManager.ts` vs `src/main/accountManager.ts`

4. **errorMsg 与 errorMessage 冗余**: tasks 表有两个错误信息字段，语义重复。

### 中优先级

5. **DevTools 生产环境开启**: `index.ts:252-256` 在生产环境也会打开 DevTools。

6. **recoverStalledTasks 未使用**: 已编写但从未调用。当前只有 `markStalledTasksAsFailed`（直接标记失败，不重试）。

7. **硬编码常量**:
   - 队列间隔 3 秒 (`publisher.ts:493`)
   - 心跳间隔 5 秒 (`index.ts:357`)
   - 僵尸任务超时 5 分钟 (`queueService.ts:61`)
   - 最大重试 3 次 (`queueService.ts:201`)
   - 水印默认坐标 `{ x: 0.905, y: 0.927, width: 0.055, height: 0.05 }` (`index.ts:77`)

### 低优先级

8. **TypeScript any 类型**: `sqliteService.ts` 中 `db` 和 `DatabaseCtor` 使用 `any` 类型，因为 `better-sqlite3` 的类型导入在 electron 环境下有兼容问题。

9. **JSON 备份冗余**: `backupService.ts` 备份的是 JSON 数据库，在 SQLite 迁移完成后这个服务应该改为备份 SQLite。
