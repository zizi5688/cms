# AI Studio 线程参考交互增强设计

## 背景

AI 工作台当前已有两套成熟交互：

- 结果卡片悬浮按钮，可把结果加入参考图或图池。
- `shellShowItemInFolder` 能力，可在 Finder 中定位到某个文件。

但线程区仍有三个缺口：

- 顶部参考图缩略图只有预览，没有“再次参考”快捷动作。
- 线程文案后面不能直接打开该线程的来源目录。
- 底部提示词输入框不能用 `Cmd+V` 粘贴图片进入参考图区。

## 目标

- 参考图缩略图 hover 时显示与结果卡片同风格的按钮，文案为“再次参考”。
- 线程文本末尾增加“打开保存文件夹”按钮，但实际打开 `run-001` 的父级来源目录。
- 提示词输入框支持粘贴剪贴板图片，并按现有导入规则进入主参考/参考图区。

## 非目标

- 不改结果卡片已有“用作参考图”行为。
- 不改 AI Studio 输入图片上限与主图/参考图排序规则。
- 不新建复杂的上传流程或独立剪贴板面板。

## 方案

### 1. 线程缩略图复用结果动作按钮

将线程头部缩略图包成 hover 容器，直接复用 `PreviewActionButton` 的样式与点击逻辑。  
按钮只做一件事：把该缩略图对应文件再次加入当前草稿任务参考输入中。

### 2. 线程按钮打开父级来源目录

为线程区增加统一的“来源目录”解析规则：

- 优先取当前线程最新输出文件路径的父目录。
- 再对该目录再取一次父目录，得到来源目录。
- 若没有输出文件，则回退到 `task.sourceFolderPath`。

这样线程按钮总是落到任务级目录，而不是具体某次 `run-*` 子目录。

### 3. 输入框支持图片粘贴

在提示词 `Textarea` 上监听 `onPaste`：

- 若剪贴板包含图片文件，阻止默认文本粘贴。
- 优先从 `clipboardData.files` 读取具备本地路径的图片文件。
- 若只有浏览器内 Blob，则转成 data URL，并通过已有 `cms.image.saveBase64` 落到临时目录，再按文件路径加入参考图区。

导入规则保持现状：

- 没有主参考时，第一张进入主参考位。
- 其余图片进入参考图列表。
- 超过上限时沿用现有忽略与提示。

## 验证标准

- 线程缩略图 hover 时可见“再次参考”，点击后当前输入区立即补入该图。
- “打开保存文件夹”会打开线程来源目录，而不是具体 `run-*` 目录。
- 在提示词输入框中按 `Cmd+V` 粘贴截图或剪贴板图片后，图片进入参考图区，文本框内容不被污染。
