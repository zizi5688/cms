# AI 单任务素材输入设计

**背景**
AI 素材工作台当前以“导入文件夹 / 片场队列”为入口，用户在单次出图场景下需要先理解任务队列与批量导入，操作路径过重。同时现有“导入文件夹”按钮缺少明确反馈，实际体验接近“按了没反应”。

**目标**
将 AI 素材工作台第一阶段收敛为“单任务出图台”：用户只需要提供 1 张主图、0~4 张参考图，即可直接在同页配置参数并发起生成。保留现有 GRSAI 提交、轮询、结果筛选链路，暂不暴露批量任务与片场队列能力。

**用户交互**

1. 用户进入 AI 素材工作台，左侧首先看到素材输入区，而不是任务队列。
2. 主图区域支持点击选图与拖拽；主图为必填。
3. 参考图区支持点击多选与拖拽追加；参考图可为空，最多 4 张。
4. 主图与参考图均展示缩略预览，并支持移除/替换。
5. 中间控制台继续配置模板、比例、数量、模型与补充要求。
6. 右侧结果区沿用现有大图/候选图筛选体验。
7. 当用户在已有结果的任务上更换输入素材时，系统先提示“将清空当前结果并重置为草稿”，确认后再覆盖输入。

**布局方案**
采用现有三栏骨架的最小改造版本：

- 左栏：从“片场”改为“素材输入”，承载主图框、参考图框与轻量操作提示。
- 中栏：保留“控制台”，只移除对“先导入文件夹”的依赖。
- 右栏：保留“结果区”，空状态文案改为围绕“添加主图后开始生成”。

**状态策略**

- 前台只聚焦 1 个当前任务。
- 若数据库中还没有 AI Studio 任务，则首次添加主图时自动创建一个草稿任务。
- 若已有当前任务，则更新它的 `primaryImagePath`、`referenceImagePaths`、`inputImagePaths`。
- 批量任务、状态筛选、队列列表在 UI 上暂时隐藏，但不删除底层能力，便于后续恢复。

**边界与约束**

- 仅接受图片文件，过滤非图片文件。
- 主图只能有 1 张；再次添加视为替换。
- 参考图可选，最多 4 张；重复文件不重复加入。
- 若已有输出结果或远端运行记录，更换输入前要求确认重置。
- 若用户取消文件选择或拖入无效文件，不应破坏当前状态。

**实现范围**

- `src/renderer/src/components/modules/AiStudio/AiStudio.tsx`
- `src/renderer/src/components/modules/AiStudio/TaskQueue.tsx`（改为素材输入面板或被替换）
- `src/renderer/src/components/modules/AiStudio/ControlPanel.tsx`
- `src/renderer/src/components/modules/AiStudio/ResultPanel.tsx`
- `src/renderer/src/components/modules/AiStudio/useAiStudioState.ts`
- 视需要复用已有 `window.electronAPI.openMediaFiles` / 扫描逻辑，不新增批量导入入口

**验证**

- `npm run typecheck`
- 手工验证：
  - 点击选择主图
  - 拖拽替换主图
  - 点击选择参考图并限制到 4 张
  - 拖拽追加参考图
  - 未设置主图时禁止生成
  - 更换已有结果任务的输入时弹出确认
