# AI 模板主提示词编辑设计

**背景**
AI 素材工作台当前已有模板表 `ai_studio_templates`，后端也会把模板的 `promptText` 与任务的 `promptExtra` 拼接后提交给 GRSAI。但前端控制台只有“模板选择”，没有“主提示词编辑 + 模板保存”入口，导致用户无法在工作台内维护自己的模板体系。

**目标**
在不改变现有 GRSAI 请求结构的前提下，为 AI Studio 增加一套“全部由用户自定义”的模板编辑体验：

1. 不再依赖任何内置系统模板。
2. 模板保存 `主提示词`。
3. 当前任务单独保存 `附加要求`。
4. 控制台内直接支持 `新建模板`、`保存`、`另存为`。
5. 生成前要求存在有效主提示词。

**核心定义**
- `模板主提示词`：保存在 `ai_studio_templates.prompt_text`。
- `附加要求`：保存在 `ai_studio_tasks.prompt_extra`，只对当前任务生效。
- 最终提交给 GRSAI 的 prompt：`模板主提示词 + 附加要求`。

**用户交互**
1. 用户进入 AI Studio，控制台顶部看到模板选择区。
2. 若当前没有模板，则下拉显示“未选择模板”，旁边可以点 `新建模板`。
3. 模板区下方直接出现两个输入区：
   - `模板名称`
   - `主提示词`
4. 模板区提供三个动作：
   - `新建模板`：清空当前模板绑定与编辑内容。
   - `保存`：若当前已绑定模板则覆盖；若未绑定模板则创建第一份模板。
   - `另存为`：始终创建新模板，并把当前任务切换到新模板。
5. 原 `要求` 改名为 `附加要求`，继续用于当前任务临时补充。
6. 点击开始生成前，若没有主图、API Key、或主提示词为空，则禁止提交。

**布局调整**
在现有中栏控制台内最小改造：
- 第一排保留 `模板 / 比例 / 数量 / 模型`。
- 模板选择控件右侧增加模板动作按钮。
- 在素材预览区上方或下方插入：
  - `模板名称` 输入框
  - `主提示词` 文本域
- 原 `要求` 文本域改名为 `附加要求`。

**状态策略**
- 模板编辑使用控制台本地草稿状态，避免每次键入都直接写数据库。
- 选中模板切换时，本地草稿同步为该模板的 `name + promptText`。
- 点击 `保存/另存为` 时才调用 `cms.aiStudio.template.upsert`。
- 保存成功后自动刷新模板列表，并将当前任务的 `templateId` 切到目标模板。
- `新建模板` 会清空当前任务的 `templateId`，进入未保存模板草稿态。

**数据与兼容性**
- 保留既有后端 `resolvePrompt()` 拼接逻辑，不改请求协议。
- 停止在前端注入 fallback 内置模板。
- 对于已有旧任务：若 `templateId` 指向不存在模板，则界面应允许用户重新选择或新建模板，而不是崩溃。

**边界与约束**
- `模板名称` 与 `主提示词` 不能为空时才允许保存。
- `开始生成` 需要有效主提示词；未保存草稿模板时不自动偷偷落库。
- 模板保存失败时保留草稿内容并给出错误提示。
- 不在本次范围内加入模板删除、模板分组、模板导入导出。

- `src/renderer/src/components/modules/AiStudio/useAiStudioState.ts`
- `src/main/services/aiStudioService.ts`（如仅复用现有接口则无需行为修改）
- `docs/plans/2026-03-07-ai-template-prompt-design.md`
- `docs/plans/2026-03-07-ai-template-prompt.md`

**验证**
- `npm run typecheck`
- 手工验证：
  - 无模板时可以新建第一份模板
  - 编辑模板名称和主提示词后可以保存
  - 另存为会生成新模板并切换当前任务
  - 切换模板会同步主提示词显示
  - 附加要求不会污染模板主提示词
  - 主提示词为空时无法开始生成
