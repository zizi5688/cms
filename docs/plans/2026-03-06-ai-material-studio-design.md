## 1. 背景与目标

Super CMS 当前已有 `ImageLab`，擅长本地素材处理、去印、超分、切图与后续工作流编排，但缺少“AI 生成素材”的前置工作台。

本方案新增一个独立模块 `AI素材工作台`，用于承接电商图生图工作流：

- 批量导入商品素材
- 维护主图、参考图、模板、自定义要求
- 调用第三方图像生成 API
- 展示价格估算与批次花费
- 允许人工筛选候选图
- 将选中结果送入现有 `ImageLab` / 后续任务链路

第一阶段的明确目标：

- 以 `GRSAI` 为服务商，调用 `nano-banana` 系列 API 完成图生图
- 支持本地文件夹导入商品
- 支持多选候选图并手动筛选
- 支持单商品与批量商品工作流
- 支持费用的单张、本商品、本批次维度展示

## 2. 非目标

第一阶段不做以下内容：

- Gemini 网页自动化或浏览器驱动方案
- 多账号 / 多 Key 轮换策略
- 自动追加系统提示词尾注
- 自动跳过人工审核直接入链路
- AI 生视频实际能力实现
- 财务级对账系统或远端账单同步

## 3. 用户输入模型

一个商品任务由以下输入组成：

- `主图`：必填，1 张
- `参考图`：可选，多张
- `模板`：从预设场景模板中选择
- `自定义要求`：可选文本
- `比例`：默认 `3:4`
- `输出张数`：默认 `1`
- `模型`：第一阶段默认 `nano-banana-2`

最终发送给模型的 prompt 只由两部分组成：

- `模板主提示词`
- `自定义要求`

系统不额外拼接任何尾注，不擅自改写用户已经稳定使用的提示词结构。

## 4. 模块边界

新增独立模块 `AI素材工作台`，不直接并入 `ImageLab`。

职责划分如下：

### AI素材工作台

负责：

- 商品导入与任务建模
- 主图 / 参考图归类
- 模板与自定义要求维护
- API 提交、轮询、结果下载
- 候选图筛选
- 费用显示
- 回流现有链路

### ImageLab

继续负责：

- 已确认素材的后处理
- 现有图片加工工作流
- 与数据工坊或下游任务的连接

这种拆分的原因是：AI 生图是完整工作流，不只是素材处理模块里的一个按钮；后续还要承接 AI 生视频，必须从一开始保留独立边界。

## 5. 界面结构

采用三段式工作台布局：

### 左侧：片场

用于展示商品任务队列：

- 商品列表
- 状态筛选
- 导入入口
- 异常项过滤入口

商品状态包括：

- `待补素材`
- `待生成`
- `排队中`
- `提交中`
- `轮询中`
- `待筛选`
- `已保留`
- `已送入链路`
- `失败（未计费）`
- `失败（已提交）`

### 中间：控制台

用于配置本次任务参数：

- 模板
- 比例
- 输出张数（默认 `1`）
- 模型
- 开始生成按钮
- 主图预览
- 参考图预览
- 自定义要求输入
- 价格摘要

控制台不承担密钥管理。

### 右侧：结果区

结果区是页面主视觉，优先级高于输入区：

- 大尺寸候选图预览
- 候选图缩略列表
- 点击图片勾选
- 多选保留 / 丢弃 / 送入链路
- 本批已发生与总预计花费展示

## 6. 视觉与交互原则

- 页面主视觉聚焦结果图，方便筛选
- 输入预览与输出预览都按竖版 `3:4` 设计
- 按钮按功能区归类：
  - 顶部仅保留全局信息
  - 控制台内放任务参数和 `开始生成`
  - 结果区内放筛选动作
- 说明性大段文本不进入主工作界面
- 候选图支持：
  - 多选
  - 点击整张图切换勾选
  - 批量保留
  - 批量送入链路
  - 批量丢弃

## 7. 主流程

### 7.1 导入

- 用户从本地导入一个或多个商品文件夹
- 一个文件夹视为一个商品任务
- 第一阶段优先支持“导入后在软件内手动归类主图与参考图”

### 7.2 配置

- 选择模板
- 填写自定义要求
- 检查主图、参考图
- 设置比例、输出张数、模型

### 7.3 生成

- 本地校验
- 提交远端任务
- 保存本地 run 记录
- 轮询结果
- 下载结果到工作区

### 7.4 筛选

- 生成成功后进入 `待筛选`
- 用户通过点击候选图进行勾选
- 支持多选

### 7.5 回流

- 仅被勾选的候选图可以送入现有链路
- 回流目标第一阶段是 `ImageLab` / 现有后续任务链路

## 8. 模板系统

模板至少包含以下字段：

- `name`
- `capability`：第一阶段为 `image`
- `base_prompt`
- `default_model`
- `default_aspect_ratio`
- `default_output_count`

模板不承载复杂结构化字段；商品级差异通过 `自定义要求` 补充。

## 9. 配置与密钥管理

`API Key` 不放在控制台中，而是放在现有 `Settings` 模块下的新分组：`AI服务`。

第一阶段的 `AI服务` 建议包含：

- 服务商：`GRSAI`
- API Key
- Base URL
- 默认模型
- 连接测试

交互约束：

- 若未配置 Key，`开始生成` 按钮置灰
- 用户点击生成时，如果未配置，应明确引导到 `Settings > AI服务`
- UI 与日志中不得回显完整 Key，仅允许掩码展示

## 11. 工作区目录结构

结果图统一落到工作区，建议目录：

- `workspace/ai-studio/tasks/<taskId>/run-001/`
- `workspace/ai-studio/tasks/<taskId>/run-002/`

每次生成尝试独立目录，避免结果互相覆盖。

建议第一阶段：

- 导入时只记录原始输入路径，不立即复制原图
- 提交或下载时按需写入工作区

## 12. API 编排

第一阶段采用“异步提交 + 轮询结果”的执行模型。

### 提交前校验

- 主图存在
- 文件格式可接受
- 模板存在
- 比例、输出张数、模型有效
- API Key 已配置

### 提交

提交成功后保存：

- 本地 task / run 标识
- 远端任务 ID
- 模型
- 提交时间
- 价格快照

### 轮询

- 定时请求结果接口
- 成功后下载结果图
- 落盘后任务进入 `待筛选`

## 13. 费用模型

第一阶段的费用展示是“价格快照估算”，不是平台账单系统。

界面展示维度：

- 单张价格
- 本商品预计 / 已发生
- 本批已发生
- 本批预计总计

费用规则：

- `预计花费 = 输出张数 × 当前模型单张价格`
- `本批预计 = 所有可生成商品预计值汇总`
- 任务提交成功后，即视为 `已发生`
- 提交前失败视为 `未计费`
- 已提交但结果异常，标记 `已提交 / 已发生` 或 `待确认`

公开价格只作为运行时快照来源，不应在代码中硬编码为长期常量。

## 14. 错误处理

建议区分以下错误：

- 本地校验失败
- 提交失败
- 远端任务异常
- 下载失败
- 回流链路失败

错误展示必须与是否计费分离，至少区分：

- `失败（未计费）`
- `失败（已提交）`

## 15. 向 AI 生视频扩展

设计中所有核心命名保持通用：

- 模块名：`AI素材工作台`
- 服务名建议：`aiStudioService`
- `tasks.capability` 未来支持 `image | video`
- `assets.role` 未来支持：
  - `reference_video`
  - `output_video`
  - `poster_frame`

未来做视频时，复用：

- 队列
- 控制台
- 费用面板
- runs 记录
- 筛选与回流动作框架

## 16. 验收标准（第一阶段）

当以下条件同时满足时，视为第一阶段可用：

- 可在 `Settings > AI服务` 配置 GRSAI 凭证
- 可导入商品文件夹并创建任务
- 可为任务设置主图、参考图、模板、自定义要求
- 可设置输出张数，默认值为 `1`
- 可调用图生图接口并轮询结果
- 可显示本商品与本批次花费
- 可对候选图进行多选勾选
- 可将所选图送入现有链路（当前落地为 `ImageLab` 一次性预填）
- 所有预览按竖版 `3:4` 呈现
- `Settings > AI服务` 的连接测试与本地配置重载可正常工作
