# 第 1 步：搭建项目骨架（Electron + React + TS + Tailwind + shadcn/ui）

## 需求理解（我会遵守）
- macOS 桌面端：Electron 主进程 + 渲染进程（React）。
- 技术栈：TypeScript、Tailwind CSS、shadcn/ui、Zustand、axios、electron-store。
- 设计基调：深色极简科技风（全局默认深色）。
- 文案规范：界面所有文字必须简体中文；代码注释也用简体中文。
- 架构目标：为后续“生产者-消费者 + Staging Area + checkpoint”模块化开发预留清晰边界。

## 脚手架选择
- 采用 `electron-vite` 作为基础工程（开发期热更新更顺滑；目录天然区分 main/preload/renderer，契合后续解耦）。

## 具体落地内容（创建/配置哪些东西）
1. 初始化 Node 工程
   - 生成 `package.json`、锁文件。
   - 增加 dev/build 脚本（开发启动 Electron + Vite；生产构建主/预加载/渲染）。

2. Electron 三件套与 React 渲染进程
   - 创建主进程入口（main）：创建 BrowserWindow、区分 dev server 与 production file 加载。
   - 创建 preload：通过 `contextBridge` 暴露最小、安全的桥接 API 占位（后续给日志/文件选择/读写 checkpoint 用）。
   - 创建 renderer（React+TS）：最小可运行页面。

3. Tailwind CSS（严格深色）
   - 安装并生成 `tailwind.config.*`、`postcss.config.*`。
   - 配置 content 扫描路径覆盖 renderer。
   - 设置全局样式：默认背景 `bg-zinc-950`、文字颜色与边框基调。

4. shadcn/ui 基础接入
   - 生成 `components.json`，配置 tailwind 与路径别名。
   - 引入 shadcn 的基础依赖（如 `clsx`、`tailwind-merge`、`class-variance-authority`、Radix 依赖等）。
   - 生成最小可用的 1-2 个基础组件（例如 Button），在首页用中文文案展示“启动成功”。

5. 项目依赖预埋（不实现业务）
   - 安装 `zustand`、`axios`、`electron-store`（先占位，后续第 3 步开始用）。

6. Git 初始化（符合你的工作流）
   - 自动初始化 Git。
   - 第一次提交只包含“脚手架 + Tailwind + shadcn 接入 + 可运行验证”。

## 验证方式（我会在第 1 步结束时做到）
- 本地启动后能打开 Electron 窗口。
- 页面为深色背景，显示中文标题与一个 shadcn Button。
- 构建命令可完整跑通（至少生成产物，不做签名/发布）。
